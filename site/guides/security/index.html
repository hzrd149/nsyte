    Security Guide¶

    This guide covers security best practices for deploying and managing nsyte sites.

    Authentication¶

    Bunker Keys¶

    Your bunker key is the primary authentication method for nsyte. Keep it secure:

    1. Storage
    2. Never commit keys to version control
    3. Use environment variables
    4. Store in secure secret management

    5. Rotate keys regularly

    6. Permissions

    7. Use minimal required permissions
    8. Create separate keys for different environments
    9. Revoke unused keys

    10. Monitor key usage

    11. Rotation

    12. Rotate keys every 90 days
    13. Use key versioning
    14. Update all environments
    15. Keep old keys for rollback

    Environment Variables¶

    # Good
export NSYTE_BUNKER_KEY="nsec1..."

# Bad
NSYTE_BUNKER_KEY="nsec1..." # In .env file

    Example GitHub Actions¶

    name: Secure Deploy

on:
  push:
    branches: [ main ]

jobs:
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Security scan
        uses: snyk/actions/node@master
        env:
          SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}

      - name: Build
        run: npm run build

      - name: Deploy
        if: success()
        env:
          NSYTE_BUNKER_KEY: ${{ secrets.NSYTE_BUNKER_KEY }}
        run: nsyte upload ./dist

    Example Monitoring Setup¶

    monitoring:
  alerts:
    - name: auth_failures
      condition: auth_failures > 5
      period: 300
      action: notify_security

    - name: relay_errors
      condition: relay_errors > 10
      period: 600
      action: notify_ops

    - name: rate_limit
      condition: rate_limit_breaches > 0
      period: 60
      action: notify_ops
